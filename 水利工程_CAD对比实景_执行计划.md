# 水利工程（大坝/水闸）CAD对比实景智能核查系统执行计划（私有化部署）

## 1. 已确认约束

- 行业场景：水利工程（优先覆盖大坝、水闸）
- 对比重点：关键几何尺寸与空间位置（长、宽、高、轴线、相对高程）
- 设计模型策略：规则获取 / 建模均可（优先满足“可对比、可核查”）
- 部署方式：私有化部署
- 风险阈值：优先采用规范；暂无明确规范项时先采用工程试运行阈值（可配置）

---

## 2. 建设目标与范围

### 2.1 总体目标
构建“以CAD设计为基准、融合无人机倾斜摄影+三维重建+AI视觉分析”的闭环系统，实现：

1. 设计—施工一致性自动核查
2. 工程隐患智能识别
3. 风险预警与辅助决策

核心目标：让图纸自动检查现实世界。

### 2.2 首期范围（MVP）
首期建议聚焦“高价值、可量化”能力：

- 大坝/水闸关键构件几何对比：长度、宽度、高程、偏移
- 典型异常识别：漏建、错建、偏移超限、局部变形疑似
- 风险分级预警：低/中/高/严重

---

## 3. 总体技术架构（私有化）

## 3.1 数据层
- 输入：CAD（DWG/DXF）、BIM/IFC（如有）、无人机影像、控制点成果
- 产物：Mesh/DOM/DSM、点云、语义构件库、差异结果库
- 存储建议：
  - 对象存储（影像/模型/切片）
  - PostgreSQL + PostGIS（语义与空间结果）
  - 时序/日志库（任务状态、推理日志）

## 3.2 处理层
- CAD解析与语义重建服务
- CAD三维化服务（规则建模优先，IFC兼容）
- 倾斜摄影重建流水线（SfM/MVS）
- 空间配准服务（控制点 + ICP）
- 差异检测服务（几何/语义/规则）
- AI识别服务（目标/缺陷/异常）
- 风险引擎服务（规则可配置）

## 3.3 应用层
- 差异检测中心
- 隐患识别中心
- 预警与工单闭环中心

## 3.4 展示层
- Cesium三维可视化
- 风险热区、分级告警、事件时间线
- 证据联动（模型位置 + 截图 + 偏差值 + 规则命中）

## 4. 分阶段执行计划（9个月，支持压缩到6个月MVP）

## 阶段A（第1-2个月）：数据与基座打通

**目标**：数据可入库、模型可重建、场景可同屏。

**任务**：
1. 制定数据标准（图层命名、构件编码、坐标基准）
2. 建立CAD解析与语义映射（坝体/闸室/闸墩/启闭机房/附属构件）
3. 建立无人机采集与重建流程
4. 完成Cesium基础展示（设计模型 + 实景模型）

**交付物**：
- 《数据标准与采集规范 v1》
- CAD语义解析服务 v1
- 倾斜摄影重建流水线 v1
- 三维场景展示原型 v1

**验收指标**：
- 样板图纸解析成功率 ≥ 90%
- 重建覆盖率 ≥ 95%（试点区域）

---

## 阶段B（第3-5个月）：配准与差异主链路

**目标**：可稳定输出“设计 vs 实景”偏差结果。

**任务**：
1. 控制点粗配准 + ICP精配准
2. 规则建模（优先满足关键构件三维表达）
3. 几何差异检测（点面距离/网格差分）
4. 语义差异检测（漏建、违建、错建）
5. 差异结果结构化（位置、偏差、证据、置信度）

**交付物**：
- 配准服务 v1（含误差评估）
- 差异检测服务 v1
- 差异结果库 v1

**验收指标**：
- 平面RMSE ≤ 10 cm（目标值）
- 高程RMSE ≤ 15 cm（目标值）
- 已知问题召回率 ≥ 80%

---

## 阶段C（第6-7个月）：AI识别与风险引擎

**目标**：从“发现差异”升级到“自动判险与建议”。

**任务**：
1. AI识别模型（构件目标、裂缝/裸露等缺陷）
2. 融合判定（AI结果 + 差异结果 + CAD语义）
3. 风险规则引擎（可配置阈值与分级）
4. 预警工单闭环（派发、复核、销项）

**交付物**：
- AI识别服务 v1
- 风险引擎 v1
- 预警管理模块 v1

**验收指标**：
- 重点类别识别精度达到试点基线
- 高风险事件预警时延 ≤ 30 分钟（批处理模式）

---

## 阶段D（第8-9个月）：试点上线与推广复制

**目标**：形成可运营、可复制的工程化产品。

**任务**：

3. 运营看板（风险热区、闭环率、整改时效）
4. 复盘与SOP沉淀

**交付物**：

## 5. 风险阈值方案（规范优先 + 试运行阈值）

说明：以下阈值先作为工程试运行版本（V0），上线后按试点项目监理/业主要求校准。

| 规则类别 | 指标 | 建议阈值（V0） | 风险等级 | 处置建议 |
|---|---|---:|---|---|
| 几何偏移 | 平面偏移 | > 0.20 m | 高 | 现场复核测量 |
| 几何偏移 | 高程偏差 | > 0.15 m | 高 | 复测并核对施工记录 |
| 尺寸偏差 | 长度/宽度偏差率 | > 2% | 中-高 | 复核放样与结构边线 |
| 语义缺失 | CAD有、实景无（漏建） | 命中即触发 | 中 | 补勘+工序核查 |
| 语义异常 | 实景有、CAD无（疑似违建） | 命中即触发 | 高 | 设计变更核查 |
| 结构异常 | 局部变形疑似（连续区域） | > 0.10 m 且面积超阈 | 严重 | 立即人工复核+专项评估 |

### 5.1 规则引擎输出示例

```json
{
  "risk_type": "sluice_axis_offset",
  "level": "high",
  "location": "E114.XXXX,N30.XXXX",
  "evidence": {
    "design_id": "SLUICE_SEG_02",
    "measured_offset_m": 0.24,
    "snapshot": "oss://evidence/task_20260416_001.png"
  },
  "suggestion": "现场复核并复测轴线控制点"
}
```

## 7. 私有化部署实施方案

## 7.1 部署拓扑（建议）
- 内网生产区：应用服务、算法服务、数据库、对象存储
- 内网隔离区：采集数据交换网关（单向或审批传输）
- 运维区：日志审计、监控告警、备份恢复

## 7.2 基础设施建议
- 容器平台：Kubernetes / K3s（按规模）
- GPU节点：用于AI推理、部分重建任务
- 数据库高可用：主从+定时快照
- 对象存储：本地化MinIO/Ceph

## 7.3 安全与合规
- 统一身份认证（AD/LDAP或OIDC）
- 最小权限访问控制（RBAC）
- 全链路审计日志（数据访问、规则变更、告警处置）
- 离线补丁与漏洞扫描机制
