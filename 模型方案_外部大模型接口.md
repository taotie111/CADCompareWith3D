# 模型方案（外部大模型接口调用）

## 1. 方案目标

在私有化部署前提下，引入“外部大模型接口”承担高语义任务，而将几何计算、配准、差异检测等确定性算法保留在内网本地执行。目标是提升：

- 风险事件解释质量
- 处置建议生成效率
- 规范问答与报告自动化能力

核心原则：**计算内生、认知外包、数据最小化出域**。

---

## 2. 职责边界（强约束）

### 2.1 本地算法（必须内网执行）
- CAD解析与三维化
- SfM/MVS重建
- 配准（GCP+ICP）
- 差异检测（几何/语义/规则）
- 风险分级判定

### 2.2 外部大模型（通过网关调用）
- 风险说明文本生成
- 整改建议草案生成
- 规范条款问答（RAG增强）
- 周报/月报文字生成

外部模型**不直接参与**风险等级最终判定。

---

## 3. 架构设计

## 3.1 LLM Gateway（内网网关）
新增统一网关服务：`llm-gateway-service`

能力：
1. 供应商抽象（provider adapter）
2. Prompt模板管理（版本化）
3. 数据脱敏与字段白名单
4. 调用审计与成本统计
5. 失败降级与多模型回退

## 3.2 调用链

1. 风险事件生成（本地）
2. 触发`/llm/risk-explain`（网关）
3. 网关提取最小必要字段并脱敏
4. 调外部LLM API
5. 返回结构化文本并落库
6. 人工复核后发布

---

## 4. 出域数据最小化规范

仅允许以下字段出域：
- 事件类型（如`sluice_axis_offset`）
- 指标值与阈值（如0.24m vs 0.20m）
- 风险等级（high/critical）
- 通用位置描述（如“闸室2段”）

禁止出域：
- 原始影像、点云、完整CAD图纸
- 精确坐标（可模糊化）
- 项目涉密名称、人员信息

---

## 5. 外部模型调用接口定义

### 5.1 风险解释
- `POST /api/v1/llm/risk-explain`

请求：
```json
{
  "event_id": "evt_xxx",
  "language": "zh-CN",
  "template_id": "risk_explain_v1"
}
```

响应：
```json
{
  "summary": "闸轴线偏移超阈值，存在施工定位偏差风险。",
  "technical_reason": "实测偏移0.24m，高于阈值0.20m。",
  "recommended_actions": ["复测控制点", "核查放样记录", "必要时局部返工"]
}
```

### 5.2 整改方案草案
- `POST /api/v1/llm/remediation-plan`

### 5.3 规范问答
- `POST /api/v1/llm/spec-qa`
- 建议先走本地RAG检索，再把检索片段提交外部模型生成答案。

---

## 6. Prompt模板策略

模板分层：
1. `system`：角色、输出约束、禁幻觉要求
2. `developer`：字段解释、语气规范、结构化输出格式
3. `user`：事件上下文数据

要求：
- 强制JSON Schema输出
- 每次输出包含“依据字段回显”
- 禁止编造规范条文编号

---

## 7. 可靠性与回退机制

1. 超时回退：主模型超时后切换备模型
2. 质量回退：输出不满足Schema时自动重试1次
3. 服务回退：外部不可用时返回本地模板文本
4. 人工兜底：high/critical必须人工复核

---

## 8. 安全与审计

- API Key托管在内网密钥系统（非明文落盘）
- 所有调用写入`llm_calls`审计表
- 保留`request_hash`用于追溯，不存敏感原文
- 配置单项目调用额度与频率限制

---

## 9. 成本控制

- 按任务类型路由模型：
  - 简单说明：轻量模型
  - 复杂建议：高性能模型
- 批量报告采用异步队列
- 启用缓存（同类事件复用摘要）

---

## 10. 供应商接入建议（抽象层）

在网关定义统一适配接口：

```text
generate(structured_prompt, schema, options) -> structured_response
```

每个供应商实现：
- `AuthAdapter`
- `CompletionAdapter`
- `ErrorAdapter`
- `UsageAdapter`

这样可以在不改业务层的前提下替换外部模型供应商。

---

## 11. 实施步骤（2周可落地）

第1周：
1. 落地`llm-gateway-service`骨架
2. 实现1个主供应商+1个备份供应商适配器
3. 打通`/llm/risk-explain`

第2周：
1. 接入审计表`llm_calls`
2. 加入脱敏与白名单
3. 接入`/llm/remediation-plan`与`/llm/spec-qa`
4. 完成人工复核工作流联调
